MVP TECHNICAL ARCHITECTURE
AMBAR BI + HELIOS IA

Este documento describe la arquitectura técnica del MVP (Minimum Viable Product) de ÁMBAR BI.

El objetivo del MVP es validar el funcionamiento del motor HELIOS para interpretar datasets y generar dashboards automáticamente.

--------------------------------------------------

1. OBJETIVO DEL MVP

El MVP debe demostrar que el sistema puede:

- cargar datasets
- interpretar columnas
- reconocer el tipo de negocio
- generar indicadores
- construir dashboards básicos

--------------------------------------------------

2. COMPONENTES PRINCIPALES

El MVP se compone de los siguientes módulos:

Data Input
Excel Reader
Data Interpreter
Dataset Profiler
Pattern Recognition
KPI Builder
Dashboard Builder

--------------------------------------------------

3. FLUJO DEL SISTEMA

El flujo del sistema será el siguiente:

Usuario carga dataset
↓
Excel Reader analiza el archivo
↓
Data Interpreter clasifica columnas
↓
Dataset Profiler identifica estructura
↓
Pattern Recognition identifica patrón de negocio
↓
KPI Builder genera indicadores
↓
Dashboard Builder genera dashboard

--------------------------------------------------

4. TECNOLOGÍAS DEL MVP

Backend

Python

Bibliotecas principales:

pandas
numpy

Procesamiento de datos:

pandas

El sistema debe ser capaz de:

leer datasets

generar indicadores

Esto permitirá validar el funcionamiento del motor HELIOS.
